# faulty-oops
Just some analysis of a page fault.
## Notes:

-Caused by trying to dereference a null pointer. `Unable to handle kernel NULL pointer dereference at virtual address 0000000000000000`
-`faulty_write+0x14/0x20 [faulty]` and `pc : faulty_write+0x14/0x20 [faulty]`
looks like faulty_write from the faulty module is the culprit. 
occured at 0x14 or 20 bytes into the program of 0x20 or 32 bytes in size. 
We can trace it back further, but we would probably start with the functions we created before diving into underlying code.
- This fault didn't cause the system to panic, but we had to log back into the program.
